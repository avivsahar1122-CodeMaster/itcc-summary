הצפנה סימטרית

📘 סוגים
  • Monoalphabetic Substitution: קבוע — נשבר בקלות ע"י Frequency Analysis.
  • Polyalphabetic: n מפתחות לסירוגין. קשה יותר לשבירה.
📐 Ciphers מודרניים
  • Stream Cipher: c(i) = m(i) ⊕ KS(i). פענוח: m(i) = c(i) ⊕ KS(i). אלגוריתם: RC4.
  • Block Cipher: בלוקים 64/128 ביט. DES=56bit (שבור), 3DES, AES=128/192/256bit (סטנדרט).
  • CBC: c(i) = K(m(i) ⊕ c(i-1)). IV = ראשוני אקראי. מונע חזרתיות.
📐 שיטות תקיפה

Brute Force | Known-Plaintext Attack | Chosen-Plaintext Attack.

הצפנה א-סימטרית — Public Key

📘 עיקרון
  • K_B+ = ציבורי (לכולם). K_B- = פרטי (רק ל-Bob).
  • K_B-(K_B+(m)) = m = K_B+(K_B-(m)) — ניתן להחליף סדר!
סימטרית (AES)
א-סימטרית (RSA)
  • מהירה מאוד
  • בעיה: חלוקת מפתח מאובטחת
  • שימוש: הצפנת גוף השיחה
  • ~100x איטי מ-AES
  • פותרת בעיית חלוקת מפתח
  • שימוש: החלפת Session Key בלבד

RSA — יצירת מפתחות ונוסחאות

📐 שלבי יצירת מפתחות RSA
  1. בחר p, q ראשוניים.
  2. n = p·q, z = (p-1)(q-1).
  3. בחר e: gcd(e,z) = 1.
  4. בחר d: (d·e) mod z = 1.
  5. K_B+ = (n,e), K_B- = (n,d).
הצפנה: c = m^e mod n פענוח: m = c^d mod n

נכון כי: (m^e mod n)^d mod n = m^(ed) mod n = m (נובע מ-(d·e) mod z = 1).

⚠ RSA vs AES — Hybrid Encryption

RSA ~100x איטי מ-AES. בפועל: RSA להחלפת Session Key בלבד, AES לתוכן.

Digital Signatures — חתימות דיגיטליות

📘 עיקרון

חתימה = הצפנה עם Private Key.

  • Bob חותם: sig = K_B-(m).
  • כולם יכולים לאמת: K_B+(sig) = m.

תכונות: Non-repudiation (לא ניתן להכחיש), Authentication, Integrity.

Certificate Authorities (CA)

📘 CA — גוף מהימן
  • CA חותמת על תעודות דיגיטליות.
  • Certificate = <Public Key + Identity> חתום ע"י CA.
  • אם סומכים ב-CA → סומכים ב-Public Key של כל מי שקיבל תעודה ממנה.

Message Integrity — Hash ו-MAC

📘 Hash Function
  • פונקציה חד-כיוונית. Many-to-one. Collision Resistance.
  • MD5=128bit (פגיע), SHA-1=160bit, SHA-256=256bit (מומלץ).
📘 MAC — Message Authentication Code

MAC = H(m + s): Hash עם מפתח סודי משותף s.

מגן מפני שינוי + זיוף זהות. אינו מגן מפני Replay Attack.

Nonce: מספר חד-פעמי → מגן מפני Playback Attack.

SSL / TLS

📐 4 שלבים
  1. Handshake: אימות Certificates, חילוף Master Secret.
  2. Key Derivation: גזירת 4 מפתחות: c_enc, c_MAC, s_enc, s_MAC.
  3. Data Transfer: כל Record = Data + MAC + Type.
  4. Connection Closure: type=FIN (סגירה מאובטחת).
⚠ Truncation Attack

תוקף מנתק TCP לפני סיום. ללא שדה type, מקבל לא יודע שהסגירה לא תקינה.

פתרון: שדה type=FIN ב-Record האחרון = מבחין בין סגירה תקינה לנתק כפוי.

בפועל: RSA לחלוקת Session Key → AES להצפנת שאר השיחה (Hybrid).
📌 צ'קליסט מבחן
  1. 4 יעדים: Confidentiality, Authentication, Integrity, Availability.
  2. AES = סימטרי, מהיר. RSA = א-סימטרי, Session Key בלבד.
  3. RSA: n=pq, z=(p-1)(q-1), de mod z=1. הצפנה: m^e mod n.
  4. Digital Signature = הצפנה עם Private Key. CA מאמתת Public Keys.
  5. Hash = חד-כיוני. MAC = Hash + מפתח. Nonce = נגד Replay.
  6. SSL: Handshake → Key Derivation → Data Transfer (type+MAC) → FIN.
  7. Truncation Attack: type=FIN מגן על סגירת SSL.
