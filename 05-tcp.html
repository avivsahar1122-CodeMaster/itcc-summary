Flow Control לעומת Congestion Control

⚠ הבחנה קריטית

Flow Control = הגנה על המקבל (rwnd — בפר פנוי).

Congestion Control = הגנה על הרשת (cwnd).

שולח מגביל ל: LastByteSent - LastByteAcked ≤ min(rwnd, cwnd)

Congestion Control — AIMD

📘 שלבי AIMD
  • Slow Start: cwnd מתחיל ב-1 MSS, מוכפל בכל RTT — עד ssthresh.
  • Congestion Avoidance: מעל ssthresh → +1 MSS לכל RTT (ליניארי).
  • Loss (Timeout): ssthresh = cwnd/2, cwnd → 1 MSS → חזרה ל-Slow Start.
  • Loss (3 dup ACKs): ssthresh = cwnd/2, cwnd = ssthresh → Fast Recovery.
📐 TCP Throughput
rate ≈ cwnd / RTT [bytes/sec] ממוצע ≈ (3/4) × W / RTT [W = חלון בזמן Loss]

3-Way Handshake

📘 למה לא Two-Way?

Two-Way: CR ישן מגיע → שרת פותח קשר ← לקוח לא יודע → שרת תקוע.

Three-Way: ACK שלישי מאשר שהלקוח עדיין רוצה את הקשר.

📐 3 שלבים
שלבמי שולחהודעה
1Client → ServerSYN (seq=x) — יוזם + ISN
2Server → ClientSYN+ACK (seq=y, ack=x+1) — ISN שרת
3Client → ServerACK (ack=y+1) — קשר מוקם

מכונות מצבים TCP

📐 Client States

CLOSED → SYN_SENT → ESTABLISHED → FIN_WAIT_1 → FIN_WAIT_2 → TIME_WAIT → CLOSED

TIME_WAIT = 2×MSL (~60 שניות). מטרה: לאפשר ל-ACK האחרון להגיע לשרת.

📐 Server States

CLOSED → LISTEN → SYN_RCVD → ESTABLISHED → CLOSE_WAIT → LAST_ACK → CLOSED

Nagle Algorithm

📘 עיקרון

בעיה: אפליקציות אינטראקטיביות שולחות Tiny-grams (כל הקשה = מנה נפרדת).

  • אם יש in-flight data → אגור עד קבלת ACK.
  • אם אין in-flight → שלח מיד.
⚠ Nagle + Delayed-ACK = Deadlock אפשרי

Nagle מחכה ל-ACK, Delayed-ACK מחכה ל-data → עיכוב. כיבוי: TCP_NODELAY.

Delay Duplicates + Wrap Around

⚠ שתי בעיות עם Sequence Numbers
  • Delay Duplicates: מנה ישנה מגיעה מאוחר עם seq שחופף מנה חדשה.
    פתרון: ISN אקראי + Timestamps + TIME_WAIT.
  • Wrap Around: TCP 32bit → מתאפס אחרי 2^32 bytes. בקצב 10 Gbps — תוך שניות!
    פתרון: PAWS (Protection Against Wrapped Sequences) — Timestamps.
📌 צ'קליסט מבחן
  1. GBN: חלון 2^k-1, Cumulative ACK, שולח מחדש k+. SR: חלון 2^(k-1), Individual ACK, k בלבד.
  2. AIMD: Slow Start (×2) → ssthresh → Avoidance (+1). Timeout → cwnd=1. 3 dup → cwnd=ssthresh.
  3. 3-Way: SYN → SYN+ACK → ACK.
  4. TIME_WAIT = 2×MSL. Flow Control = rwnd. Congestion Control = cwnd.
  5. Nagle + Delayed-ACK = deadlock. כיבוי: TCP_NODELAY.
