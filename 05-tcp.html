פרוטוקול TCP ואמינות נתונים

Sliding Window, כותרת TCP, Flow Control, Congestion Control ו-3-Way Handshake

זה הפרק הכבד ביותר בקורס. למבחן: מנגנון Sliding Window ו-Pipelining, שדות כותרת TCP, Fast Retransmit (3 dup ACKs), AIMD — Slow Start ו-Congestion Avoidance, ו3-Way Handshake.

שידור חוזר (Retransmission)

📘 שני מנגנונים
  • Timeout דינמי: מחושב לפי ממוצע נע של RTT.
  • Fast Retransmit: שידור חוזר מיידי ב-3 Duplicate ACKs — ללא המתנה ל-Timeout.
📐 נוסחת EstimatedRTT
EstimatedRTT = (1-α)·EstimatedRTT + α·SampleRTT

בדרך כלל α = 0.125

Flow Control — בקרת זרימה

📘 Speed Matching

מניעת הצפת חוצץ המקבל. השולח מגביל את עצמו לשדה rwnd בכותרת TCP.

📐 נוסחת ניהול שידור
LastByteSent - LastByteAcked ≤ rwnd

Congestion Control — AIMD

📘 עקרון AIMD
  • Slow Start: גידול מעריכי של cwnd — הכפלה בכל RTT, עד ssthresh.
  • Congestion Avoidance: גידול ליניארי (MSS אחד בכל RTT) לאחר חציית ssthresh.
  • Loss (Timeout): cwnd → 1 MSS, ssthresh = cwnd/2 — חזרה ל-Slow Start.
  • Fast Retransmit (3 dup ACKs): cwnd = ssthresh = cwnd/2 — מעבר ישיר ל-Congestion Avoidance.
⚠ חשוב להבחין

Flow Control = הגנה על המקבל (rwnd). Congestion Control = הגנה על הרשת (cwnd). שניהם פועלים במקביל; השולח מגביל ל-min(rwnd, cwnd).

3-Way Handshake

📐 לחיצת יד משולשת — שלושה שלבים
שלבשולחהודעהמטרה
1Client → ServerSYN (seq=x)יוזם קשר, מגדיר ISN
2Server → ClientSYN-ACK (seq=y, ack=x+1)אישור + ISN שרת
3Client → ServerACK (ack=y+1)קשר מוקם
📘 סיום קשר — FIN/TIME_WAIT

כל צד שולח FIN לסגירת כיוון שלו. הצד המסיים ראשון עובר למצב TIME_WAIT (~30 שניות) להבטחת קבלת ה-ACK האחרון.

📌 צ'קליסט מבחן
  1. Fast Retransmit = 3 dup ACKs; Timeout = cwnd→1.
  2. AIMD: Slow Start (מעריכי) → ssthresh → Congestion Avoidance (ליניארי).
  3. 3-Way Handshake: SYN → SYN-ACK → ACK.
  4. Flow Control: rwnd (הגנה על מקבל). Congestion Control: cwnd (הגנה על רשת).
