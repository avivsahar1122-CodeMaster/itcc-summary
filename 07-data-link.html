שכבת עורק הנתונים — Data Link Layer

MAC, ARP, CSMA/CD, מסגרת Ethernet, Switch לעומת Hub

פרק זה עוסק בהעברה בין צמתים סמוכים. למבחן: כתובת MAC וARP, ALOHA לעומת CSMA/CD לעומת CSMA/CA עם יעילויות, שדות מסגרת Ethernet, וSwitch לעומת Hub.

כתובות MAC ו-ARP

📘 כתובת MAC
  • כתובת פיזית בת 48 ביטים (6 בתים, Hex)
  • קבועה וצרובה על כרטיס הרשת (NIC)
  • לדוגמה: 00:1A:2B:3C:4D:5E
📘 ARP — Address Resolution Protocol

מתרגם כתובת IP → MAC מקומית באמצעות Broadcast ברשת המקומית. התגובה מוחזרת Unicast לשואל.

הבחנה: DNS מתרגם שם → IP. ARP מתרגם IP → MAC. שתי שכבות שונות לחלוטין.

⚠ CSMA/CD לעומת CSMA/CA

ב-Ethernet קווי — CD (Collision Detection): מגלה התנגשות ושולח Jam Signal + Exponential Backoff.
ב-Wi-Fi — CA (Collision Avoidance): לא ניתן לגלות התנגשות (כי השידור שלך מסתיר אותה), לכן מנסים למנוע מראש.

רכיבי הרשת המקומית

Hub (רכזת) — שכבה 1
Switch (מתג) — שכבה 2
  • משכפל אות לכל היציאות
  • מרחב התנגשות יחיד
  • אין אינטליגנציה
  • נדיר כיום
  • לומד כתובות MAC ומנתב ל-port מדויק
  • Traffic Isolation — ללא התנגשויות
  • Buffering + Full-Duplex
  • נפוץ בכל רשת מקומית
📘 Spanning Tree Protocol (STP)

מניעת לולאות לוגיות ב-Switch: בוחר עץ פורש (Spanning Tree) — חוסם חלק מהקישורים. פרוטוקול מודרני: RSTP (Rapid STP).

📌 צ'קליסט מבחן
  1. MAC = 48 ביט; ARP = IP→MAC ב-Broadcast.
  2. Slotted ALOHA = 37%; ALOHA = 18%.
  3. CSMA/CD = Ethernet (גילוי); CSMA/CA = Wi-Fi (הימנעות).
  4. Switch לומד MAC ומנתב; Hub משדר לכולם.
