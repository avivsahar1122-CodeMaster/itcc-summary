שכבת התובלה — UDP

Sockets, Multiplexing, כותרת UDP, Checksum, Endianness

פרק זה פותח את שכבת התובלה. למבחן: Multiplexing ו-Demultiplexing ב-UDP לעומת TCP, מבנה כותרת UDP (8 בייט), Checksum — 1's complement, וEndianness.

Multiplexing & Demultiplexing

📘 הגדרות
  • Multiplexing (שולח): אוסף נתונים מכמה Sockets → מוסיף כותרת → שולח.
  • Demultiplexing (מקבל): מפנה Segment ל-Socket הנכון לפי כותרת.
📐 UDP לעומת TCP Demux
  • UDP (Connectionless): Socket = (Dst IP, Dst Port). Src אינו קובע.
  • TCP (Connection-oriented): Socket = 4-tuple: (Src IP, Src Port, Dst IP, Dst Port).
⚠ הבדל קריטי

UDP: Dst Port מספיק לניתוב ל-Socket.

TCP: נדרש 4-tuple שלם — שני לקוחות על אותו Dst Port = Sockets שונים.

טווחי פורטים

📐 IANA Port Ranges
  • 0–1023: Well-Known — HTTP=80, HTTPS=443, DNS=53, SSH=22, FTP=21.
  • 1024–49151: Registered Ports.
  • 49152–65535: Dynamic/Ephemeral — OS מקצה אוטומטית.

פרוטוקול UDP

📘 מאפייני UDP
  • Connectionless — אין Handshake.
  • Best Effort — אין הבטחת אמינות, אין סדר מובטח.
  • אין Congestion Control או Flow Control.
  • שימושים: DNS, Streaming, VoIP, Gaming — כשמהירות > אמינות.
📐 כותרת UDP — 8 בייט בלבד (4 שדות × 2B)
שדהגודלתפקיד
Source Port16 ביטפורט מקור
Destination Port16 ביטפורט יעד
Length16 ביטHeader + Data בבייטים. מינימום = 8.
Checksum16 ביטגילוי שגיאות. Optional ב-IPv4, Mandatory ב-IPv6.

Checksum — גילוי שגיאות

📘 1's Complement

מחשבים משלים ל-1 של סכום כל המילים ב-Segment.

מקבל מחשב סכום — אם התוצאה ≠ 0xFFFF, זוהתה שגיאה. גילוי בלבד — לא תיקון.

💡 דוגמה
מילה 1: 0110011001100110 מילה 2: 0101010101010101 סכום: 1011101110111011 Checksum: 0100010001000100 (complement)

Endianness

📐 Network Byte Order = Big Endian
  • htons() / htonl() — המרה מ-Host לרשת (Short/Long).
  • ntohs() / ntohl() — המרה מרשת ל-Host.

TCP לעומת UDP

TCP
UDP
  • אמין — הבטחת מסירה
  • מבוסס קישור (3-Way Handshake)
  • Flow Control + Congestion Control
  • HTTP, FTP, Email
  • לא אמין — Best Effort
  • Connectionless
  • אין בקרה — מהיר, overhead נמוך
  • DNS, VoIP, Streaming, Gaming
📌 צ'קליסט מבחן
  1. UDP = 4 שדות × 16 ביט = 8 בייט כותרת.
  2. UDP Demux: Dst Port בלבד. TCP Demux: 4-tuple.
  3. Checksum = 1's complement; גילוי בלבד, לא תיקון.
  4. Big Endian = Network Byte Order. htons() לפני שליחה.
