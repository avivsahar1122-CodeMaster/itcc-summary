מבוא ומודל השכבות

OSI, TCP/IP, אנקפסולציה ועקרונות ארכיטקטורת שכבות

זה הפרק הבסיסי של הקורס. למבחן: 5 שכבות TCP/IP והPDU של כל אחת, אנקפסולציה, סוגי תווך פיזי, Throughput ו-Bottleneck, ו5 סוגי איומי אבטחה.

מודלי שכבות — OSI ו-TCP/IP

📘 שני המודלים
  • OSI: 7 שכבות — תיאורטי, כולל שיחה, ייצוג, אפליקציה.
  • TCP/IP: 5 שכבות — מעשי, ממזג את ה-3 העליונות לשכבת Application אחת.
למבחן: OSI = 7 שכבות (תיאורטי). TCP/IP = 5 שכבות (מעשי).

אנקפסולציה

📘 תהליך האנקפסולציה

בשליחה: כל שכבה מוסיפה Header. בקבלה: כל שכבה מסירה Header.

  • Application → Message
  • Transport + Header → Segment
  • Network + Header → Datagram
  • Data Link + Header → Frame
  • Physical → Bits

תווך פיזי (Physical Media)

📘 סוגי תווך
  • Twisted Pair (UTP/STP): נפוץ ביותר, עד 10 Gbps.
  • Coaxial Cable: מוקף מגן, שימוש ישן ברשתות / כבל טלוויזיה.
  • Fiber Optic: שידור אור, עד Tbps, חסין להפרעות אלקטרומגנטיות.
  • Wireless: ערוץ משותף (Half-Duplex בד"כ): WiFi, סלולרי, WiMax.

Throughput ו-Bottleneck

📐 נוסחה
Throughput = min(Rs, Rc) [bits/sec] זמן העברת קובץ F: T = F / min(Rs, Rc)

Bottleneck = הקישור עם ה-bandwidth הנמוך ביותר בנתיב.

איומי אבטחה בסיסיים

📘 5 סוגי איומים
  • Malware: Virus (דרך משתמש), Worm (עצמאי ברשת), Trojan (מוסווה).
  • Sniffing: האזנה פסיבית לתעבורה — קריאת מידע שעובר.
  • Spoofing: זיוף כתובת מקור (IP/MAC) להתחזות למשתמש אחר.
  • DoS / DDoS: הצפת שרת. DDoS = מגיע מכמה מקורות בו-זמנית.
  • Man-in-the-Middle: יירוט ושינוי תקשורת בין שני צדדים.
