IP Fragmentation

📘 מתי ולמה?

כשגודל Datagram > MTU של הקישור, הנתב מפצל ל-Fragments. Reassembly = ביעד בלבד.

  • ID: כל Fragments של אותה מנה = אותו ID.
  • MF=1: יש עוד Fragments. MF=0 רק בפרגמנט האחרון.
  • DF=1: אסור לפצל. אם MTU קטן → ICMP "Fragmentation Needed".
📐 נוסחות Fragmentation
Data per fragment = MTU − 20 Fragment Offset = byte_position / 8

IP Addressing — CIDR ו-Subnets

📘 הגדרות
  • Subnet: קבוצת ממשקים עם אותו prefix. תקשורת ישירה — ללא נתב.
  • /x (CIDR): x ביטים Network, (32-x) ביטים Host → 2^(32-x) כתובות.

DHCP — תהליך DORA

📐 4 שלבים
  1. Discover: src=0.0.0.0 → dst=255.255.255.255 (Broadcast) — "מי יכול לתת IP?"
  2. Offer: שרת מציע IP + lease time.
  3. Request: לקוח מבקש (עדיין Broadcast — יכולים להיות כמה שרתים).
  4. ACK: שרת מאשר.

מה מקבלים: IP, Subnet Mask, Default Gateway, DNS Server.

NAT — Network Address Translation

📘 עיקרון

נתב NAT: IP פנימי + Port → IP ציבורי אחד + Port ייחודי. שומר NAT Table.

Private ranges: 10.x, 172.16-31.x, 192.168.x.x.

  • יתרונות: חיסכון בכתובות, אבטחה בסיסית.
⚠ חסרונות NAT
  • שובר End-to-End — לא ניתן להתחבר מבחוץ ישירות.
  • בעיה עם SIP/VoIP — IP מוטמע ב-Payload.
  • Stateful — NAT Table = bottleneck פוטנציאלי.

Port-Forwarding: גישה מבחוץ לשרת פנימי — הגדרה ידנית.

אלגוריתמי ניתוב — AS, RIP, OSPF

📘 Autonomous System (AS)
  • AS: קבוצת נתבים תחת ניהול אחד (ISP, אוניברסיטה).
  • Intra-AS: בתוך AS. מטרה: ביצועים. פרוטוקולים: RIP, OSPF.
  • Inter-AS: בין AS-ים. מטרה: מדיניות. פרוטוקול: BGP.

RIP Split Horizon: פתרון חלקי ל-Count-to-Infinity — לא שולח מידע חזרה לשכן שממנו קיבלת.

IPv6 — מבוא

📘 למה IPv6?
  • IPv4: 32 bit ≈ 4 מיליארד כתובות → אזלו ב-2011.
  • IPv6: 128 bit = 2^128 כתובות. Header = 40B קבוע (ללא Options, ללא Checksum).
  • מעבר — Tunneling: עוטפים IPv6 packet ב-IPv4 Datagram בין Routers.
📌 צ'קליסט מבחן
  1. Forwarding = data plane (מהיר). Routing = control plane (חישוב).
  2. Fragmentation: Offset = byte/8. MF=1 בכל Fragment חוץ מאחרון.
  3. DHCP: Discover→Offer→Request→ACK. מקבלים: IP, Mask, GW, DNS.
  4. NAT: חוסך כתובות, שובר End-to-End, בעיית SIP.
  5. ICMP: ping=Type8/0, traceroute=Type11/0, TTL Exceeded, Frag Needed.
  6. RIP = Bellman-Ford, 30s, 15 hops. OSPF = Dijkstra, Triggered, Areas.
  7. IPv6 = 128 bit, Tunneling למעבר.
