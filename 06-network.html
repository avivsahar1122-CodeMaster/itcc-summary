שכבת הרשת — Network Layer

IP, CIDR, DHCP, NAT, Fragmentation ואלגוריתמי ניתוב

פרק זה עוסק בשכבת הרשת — שם ה-IP מנתב מנות מקצה לקצה. למבחן: Forwarding לעומת Routing, CIDR ומסיכות רשת, DHCP 4 שלבים, NAT, חישוב Fragment Offset, וRIP לעומת OSPF.

Forwarding לעומת Routing

Forwarding (העברה)
Routing (ניתוב)
  • תהליך חומרתי, מקומי בנתב
  • מעביר מנה מכניסה ליציאה
  • מהיר מאוד (nanoseconds)
  • תהליך תוכנתי — חישוב מסלולים
  • בונה את טבלאות ההעברה
  • איטי יותר (seconds-minutes)

כתובות IP וארכיטקטורה

📘 IPv4

כתובת IPv4 = 32 ביטים. מורכבת מ-Subnet ID (זהה רשת) ו-Host ID (זהה מארח).

📐 CIDR Notation
/24 = 255.255.255.0 → 24 ביטי רשת, 8 ביטי מארח = 256 כתובות
  • כתובת רשת: כל ביטי ה-Host = 0
  • Broadcast: כל ביטי ה-Host = 1
  • כתובות שמישות: גודל הרשת − 2

DHCP — חלוקת כתובות אוטומטית

📐 4 שלבי DHCP
Discover → Offer → Request → ACK
  1. Discover: הלקוח שולח Broadcast — מחפש שרת DHCP
  2. Offer: השרת מציע כתובת IP
  3. Request: הלקוח מבקש את הכתובת
  4. ACK: השרת מאשר את ההקצאה

NAT — Network Address Translation

📘 NAT

פותר מחסור בכתובות IPv4. כל הרשת הביתית משתמשת בכתובת ציבורית אחת. הנתב מתרגם IP פנימי + פורט → IP ציבורי + פורט ייחודי (NAT Table).

⚠ ביקורת על NAT

NAT מפר את עקרון end-to-end של האינטרנט — מכשירים פנימיים אינם ניתנים לגישה ישירה מבחוץ.

Fragmentation — פיצול מנות

📘 מתי מפצלים?

כאשר גודל ה-Datagram חורג מה-MTU של הקו. ההרכבה מתבצעת ביעד בלבד.

📐 שדות Fragmentation
  • Identifier (16 ביט): מזהה את כל קבוצת השברים
  • MF (More Fragments): 1 = יש עוד שברים; 0 = שבר אחרון
  • Fragment Offset: מיקום השבר בתוך ה-Datagram המקורי, ביחידות של 8 בתים
Fragment Offset = מיקום בתים / 8

אלגוריתמי ניתוב

Distance Vector — RIP
Link State — OSPF
  • כל נתב שומר וקטור מרחקים
  • שולח לשכנים בלבד
  • בעיה: Count to Infinity Loops
  • פשוט, איטי להתאים
  • כל נתב מכיר את המפה המלאה (flooding)
  • חישוב מסלול קצר ביותר — Dijkstra
  • אין לולאות
  • מהיר, תומך Areas היררכיים
📌 צ'קליסט מבחן
  1. CIDR: /24 = 256 כתובות = 254 שמישות.
  2. DHCP: Discover→Offer→Request→ACK.
  3. Fragment Offset = מיקום/8 (יחידות של 8 בתים).
  4. RIP = Distance Vector; OSPF = Link State + Dijkstra.
